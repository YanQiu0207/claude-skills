---
name: md-fmt
description: |
  对 Markdown 文件进行一站式标准化：先排版规范化，再网络图片本地化。当用户要求"规范化 markdown"、"标准化 markdown"、"离线化 markdown"、"本地化 markdown 图片并整理排版"时触发。只要用户提到对 markdown 做排版整理和图片下载的组合操作，就应该使用这个技能，即使用户没有明确说"标准化"。
  支持单文件和多文件批量处理，多文件时自动并行。
---

当此 skill 生效时，回答第一行固定写：Using skill: md-fmt

## 判断处理模式

根据用户输入判断文件数量：
- **单文件**：用户提供了 1 个文件 → 走「单文件处理流程」
- **多文件**：用户提供了多个文件路径或使用通配符（如 `*.md`、`**/*.md`）→ 走「多文件并行处理」

---

## 多文件并行处理

当检测到需要处理 2 个及以上文件时，使用 subagent 并行处理：

### 步骤 1：确定文件列表

- 如果用户给了通配符，用 Glob 工具展开为具体文件路径列表
- 如果用户给了多个文件名，逐个确认路径存在

### 步骤 2：并行分发

在**同一条消息**中，为每个文件发起一个 Agent 工具调用，实现真正的并行执行：

- 使用 `general-purpose` agent（不设 subagent_type，即默认值）
- 使用 `model: "sonnet"` 以获得更快的处理速度
- prompt 中必须写明文件的**绝对路径**（subagent 是独立上下文，看不到主对话）
- prompt 中必须包含完整的处理指令（两个阶段的详细步骤）
- 建议同时并行不超过 8 个文件，超过时分批处理

每个 Agent 的 prompt 模板如下：

```
你需要对文件 {绝对路径} 执行 Markdown 标准化处理，分两个阶段：

**阶段 1：排版规范化**
1. 使用 Skill 工具调用 `md-zh` 技能加载中文排版规则
2. 读取目标文件全文
3. 按照 md-zh 规则修改文件内容（中英文间加空格、中文与数字间加空格、全角标点、专有名词大小写等）
4. 将修改后的内容写回原文件

**阶段 2：图片本地化**
1. 使用 Skill 工具调用 `md-img-local` 技能
2. 按照该技能流程，扫描文件中的网络图片链接，下载到 assets 目录，替换为本地相对路径

处理完成后，返回简洁的结果摘要：
- 文件路径
- 排版：修改了哪些方面，或"无需修改"
- 图片：成功 N 张 / 失败 M 张，或"无网络图片"
- 失败链接（如有）
```

示例（处理 3 个文件时，在一条消息中同时发起 3 个 Agent 调用）：

```
Agent({
  description: "md-fmt: file1.md",
  model: "sonnet",
  prompt: "你需要对文件 E:/work/blog/ai/file1.md 执行 Markdown 标准化处理……（完整 prompt）"
})
Agent({
  description: "md-fmt: file2.md",
  model: "sonnet",
  prompt: "你需要对文件 E:/work/blog/ai/file2.md 执行 Markdown 标准化处理……（完整 prompt）"
})
Agent({
  description: "md-fmt: file3.md",
  model: "sonnet",
  prompt: "你需要对文件 E:/work/blog/ai/file3.md 执行 Markdown 标准化处理……（完整 prompt）"
})
```

### 步骤 3：汇总报告

所有 agent 返回后，汇总每个文件的处理结果，格式：

| 文件 | 排版 | 图片 | 备注 |
|------|------|------|------|
| file1.md | 已修改 | 3 成功 / 0 失败 | |
| file2.md | 无需修改 | 无网络图片 | |
| file3.md | 已修改 | 2 成功 / 1 失败 | 失败: http://... |

## 单文件处理流程

对用户提供的单个 Markdown 文件，严格按以下顺序执行两个阶段：

### 阶段 1：排版规范化

使用 Skill 工具调用 `md-zh` 技能，对目标文件应用中文排版规范。具体做法：

1. 读取目标 Markdown 文件全文
2. 调用 `md-zh` 技能加载排版规则
3. 按照 md-zh 的规则逐项检查并修改文件内容，包括但不限于：
   - 中英文之间加空格
   - 中文与数字之间加空格
   - 使用全角中文标点
   - 专有名词大小写
   - 标点符号规范
4. 将修改后的内容写回原文件
5. 告知用户排版规范化已完成

### 阶段 2：图片本地化

使用 Skill 工具调用 `md-img-local` 技能，将文件中的网络图片下载到本地。具体做法：

1. 调用 `md-img-local` 技能
2. 按照该技能的流程，扫描文件中的网络图片链接，下载到 assets 目录，替换为本地相对路径
3. 告知用户图片本地化已完成，报告下载结果

### 完成报告

两个阶段都完成后，向用户汇总：
- 排版修改了哪些方面
- 下载了多少张图片、保存位置
- 如有图片下载失败，列出失败的链接

---

## 边界情况

- **文件中没有网络图片**：正常执行阶段 1，阶段 2 告知用户无需下载图片
- **文件排版已经规范**：阶段 1 告知用户无需修改，继续执行阶段 2
- **用户只想做其中一项**：如果用户只要排版不要图片（或反过来），引导用户直接使用对应的单独技能（`md-zh` 或 `md-img-local`）
- **通配符匹配到 0 个文件**：告知用户没有匹配到文件，请检查路径
- **文件数超过 8 个**：分批处理，每批最多 8 个并行
