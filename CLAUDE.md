\# 全局指令

\## 沟通方式
\- 使用中文回复

\## 沟通要求
\- **事实性内容必须有来源支撑**：回答或写入任何事实性内容（工具行为、API、框架特性、技术细节等）时，必须有可靠来源（官方文档、WebFetch 确认、实测结果）；若不确定，明确说明"不确定"或"需要确认"，不能凭推断或训练数据中的模糊印象给出确定性的陈述

\## 通用代码风格
\- 缩进使用 4 空格

\## Claude Code 参考知识库
\- 参考文件：`~/.claude/claude_ref/claude-code-guide.md`
\- **查询时**：回答 Claude Code 相关问题时
  \- 模型自身有明确答案：直接回答；若此文件中缺少该内容，询问用户是否更新到此文件
  \- 模型自身没有明确答案：查阅此文件作为补充；若参考了此文件内容，要告知用户信息来源
\- **记录时**：当用户要求"记录"某个知识点时
  \- Claude Code 相关内容 → 写入此文件
  \- 其他内容 → 先询问用户写入位置（某个具体文件、项目记忆、还是全局记忆）

\## 中文 Markdown 排版规范
\- 规范技能：`md-zh`（使用 Skill 工具调用以获取完整规则）

\## 记忆与知识写入规则

\- **行为规则的持久化位置**：
  \- 全局适用（所有项目都应遵守）→ 写入 `~/.claude/CLAUDE.md`
  \- 当前项目专属 → 写入项目记忆
  \- 默认优先考虑全局，除非规则明显只与某个项目相关

\## 创建技能的规则
\- 创建或修改技能时，先调用 `cc-adv-guide` 获取设计规范
