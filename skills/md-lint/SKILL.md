当此 skill 生效时，回答第一行固定写：Using skill: md-lint

> **⚠ 并发安全**：本技能被 `batch-md-lint` 通过多个并行后台 agent 同时调用，每个 agent 处理不同文件。修改本技能时，必须确保不引入共享状态（如全局临时文件、固定名称的中间产物等），否则并发执行会产生冲突。

> **注意**：本技能仅处理单个文件。如果用户提供了多个文件路径或通配符（如 `*.md`、`**/*.md`），请告知用户并改为调用 `batch-md-lint` 技能。

## 输入

用户提供一个**目标文件路径**（待检查的 Markdown 文件）。

排版规范参见 `md-zh` 技能。使用 Skill 工具调用 `md-zh` 获取完整规则内容。

## 处理流程

### 1. 读取目标文件

使用 Read 工具读取目标文件全文。

### 2. 逐条检查并记录问题

对照 `md-zh` 技能中的每一条规则，逐行扫描目标文件，记录所有违规之处。每个问题记录：

- **行号**
- **规则**：违反了哪条规则（简短描述）
- **原文**：有问题的原始文本片段
- **修正**：修复后的正确文本片段

**跳过以下区域，不做检查**：
- YAML frontmatter（`---` 包裹的头部元数据）
- 代码块（`` ``` `` 包裹的内容）
- 行内代码（`` ` `` 包裹的内容）
- HTML 标签及其属性
- URL 链接地址（`[text](url)` 中的 url 部分，以及裸链接）
- 图片路径（`![alt](path)` 中的 path 部分）

### 3. 自动修复

使用 Edit 工具修复目标文件中的所有问题。同一行或相邻行的问题可合并为一次 Edit。修复只调整排版格式，不改变语义内容。

### 4. 输出总结

修复完成后，输出检查报告：

```
## 检查报告：{文件名}

**规范**：markdown-zh
**问题总数**：{N} 处（已全部修复）

### 问题明细

| # | 行号 | 规则 | 原文 | 修正 |
|---|------|------|------|------|
| 1 | 12 | 中英文之间加空格 | `在LeanCloud上` | `在 LeanCloud 上` |
| ... | ... | ... | ... | ... |

### 按规则分类统计

| 规则 | 数量 |
|------|------|
| 中英文之间加空格 | 5 |
| ... | ... |
```

如果文件完全符合规范，输出：

```
## 检查报告：{文件名}

**规范**：markdown-zh

文件完全符合排版规范，无需修改。
```
