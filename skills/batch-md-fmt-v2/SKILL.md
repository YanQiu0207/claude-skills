## 阶段 1：排版规范化

### 中文 Markdown 排版规范

**空格规则**：

1. 中英文之间需要增加空格
   - 正确：在 LeanCloud 上，数据存储是围绕 AVObject 进行的。
   - 错误：在LeanCloud上，数据存储是围绕AVObject进行的。
   - 例外：「豆瓣FM」等产品名词按官方格式书写。

2. 中文与数字之间需要增加空格
   - 正确：今天出去买菜花了 5000 元。
   - 错误：今天出去买菜花了5000元。

3. 数字与单位之间需要增加空格
   - 正确：光纤有 10 Gbps，SSD 有 20 TB
   - 错误：光纤有 10Gbps，SSD 有 20TB
   - 例外：度数／百分比与数字之间不加空格（90°、15%）

4. 全角标点与其他字符之间不加空格
   - 正确：刚刚买了一部 iPhone，好开心！
   - 错误：刚刚买了一部 iPhone ，好开心！

**标点规则**：

5. 不重复使用标点符号
   - 正确：德国队竟然战胜了巴西队！
   - 错误：德国队竟然战胜了巴西队！！

**全角半角规则**：

6. 使用全角中文标点
   - 正确：嗨！你知道嘛？今天前台的小妹跟我说「喵」了哎！
   - 错误：嗨! 你知道嘛? 今天前台的小妹跟我说 "喵" 了哎！
   - 例外：英文句子内使用半角标点。

7. 数字使用半角字符
   - 正确：这个蛋糕只卖 1000 元。
   - 错误：这个蛋糕只卖 １０００ 元。

**名词规则**：

8. 专有名词使用正确的大小写
   - 正确：使用 GitHub 登录
   - 错误：使用 github 登录 / 使用 GITHUB 登录 / 使用 Github 登录

9. 不使用不地道的缩写
   - 正确：熟悉 TypeScript、HTML5，理解 React、Next.js
   - 错误：熟悉 Ts、h5，理解 RJS、nextjs

**Markdown 格式规则**：

10. 加粗标记不包裹标点
    - 正确：**注意**：这会丢弃工作区的修改。
    - 错误：**注意：** 这会丢弃工作区的修改。/ **注意：**这会丢弃工作区的修改。

### 排版操作步骤

1. 使用 Read 工具读取目标文件全文
2. 对照上述每条规则，逐行扫描，记录所有违规（行号、规则、原文、修正）
3. 使用 Edit 工具修复所有问题（同一行或相邻行的问题可合并为一次 Edit）
4. 修复只调整排版格式，不改变语义内容

**跳过以下区域，不做检查**：
- YAML frontmatter（\`---\` 包裹的头部元数据）
- 代码块（\`\`\` 包裹的内容）
- 行内代码（\` 包裹的内容）
- HTML 标签及其属性
- URL 链接地址（\`[text](url)\` 中的 url 部分，以及裸链接）
- 图片路径（\`![alt](path)\` 中的 path 部分）

## 阶段 2：图片本地化

### 图片本地化操作步骤

1. 读取文件，提取所有 \`![alt](url)\` 格式中以 HTTP/HTTPS 开头的图片 URL，去重
2. 如无网络图片，报告"无网络图片"，跳过此阶段
3. 检查文件同级目录下是否存在 \`assets\` 目录，不存在则创建
4. 提取 Markdown 文件名（去除特殊字符）作为图片前缀
5. 遍历所有唯一图片 URL：
   - 生成本地文件名：\`[前缀]_[序号]_[原文件名]\`
   - 使用 Bash 工具下载图片到 assets/ 目录，失败重试最多 2 次
   - 记录成功/失败结果
6. 使用 Edit 工具将 Markdown 中每个成功下载的网络图片链接替换为对应的本地相对路径
   - 只替换图片链接，不修改其他内容
   - 下载失败的图片保留原链接

### 约束

- 不得修改 Markdown 文件的其他内容
- 不得覆盖本地已有的同名文件（添加后缀避免冲突）
