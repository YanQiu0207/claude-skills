## 存储位置

所有交接文件统一存放在 `~/.claude/handoffs/` 目录下，该目录跨会话、跨项目共享。

每个文件以用户指定的 filename 命名，自动添加 `.md` 后缀（如用户已提供后缀则不重复添加）。

## 脱敏规则

写入交接文件前，必须对内容进行脱敏处理：

- **文件路径**：将操作系统绝对路径（如 `C:\Users\<username>\...` 或 `/home/<username>/...`）替换为 `~` 开头的通用路径（如 `~/.claude/...`）
- **用户名**：路径中的系统用户名不得出现在文件内容中，用 `~` 替代
- **其他 PII**：如有其他个人信息，一并匿名化处理

## 工具调用权限

以下操作**无需用户确认**，直接执行：

- 搜索/扫描 `~/.claude/handoffs/` 目录（Glob）
- 新建文件（目标文件不存在时）
- 读取文件内容（Read）

## 命令

### `/handoff write <filename>`

将当前对话的上下文总结写入交接文件。

#### 处理流程

1. **解析 filename**：取用户提供的 filename，若不以 `.md` 结尾则自动补上 `.md`
2. **总结当前对话**：回顾整个对话历史，生成一份结构化的交接摘要，包含：
   - **背景**：用户要解决什么问题 / 达成什么目标
   - **已完成的工作**：做了哪些事，关键改动列表（含文件路径）
   - **当前状态**：进展到什么阶段，是否有未完成的部分
   - **遗留问题 / 待办**：还有什么需要后续处理的
   - **关键决策与原因**：对话中做出的重要决策及其理由
   - **注意事项**：容易踩坑的点、需要特别关注的地方
3. **对摘要内容进行脱敏**（参见脱敏规则）
4. **检查文件是否已存在**：
   - **文件不存在**：直接创建并写入，无需用户确认
   - **文件已存在**：
     a. 读取已有内容
     b. 将当前对话摘要与已有内容进行对比
     c. **如果内容关联度较高且无冲突**（例如是同一任务的后续进展）：直接追加或合并更新，不需要用户确认
     d. **如果内容关联度较低或存在冲突**：向用户展示差异，说明冲突点，询问用户选择：
       - **覆盖**：用新内容替换旧内容
       - **追加**：在末尾追加新内容，保留旧内容
       - **取消**：不写入
5. **写入文件**：将脱敏后的摘要写入 `~/.claude/handoffs/<filename>.md`
6. **输出确认**：告知用户文件已写入，显示文件路径

#### 交接文件格式

```markdown
# 交接记录：<简短标题>

> 创建时间：YYYY-MM-DD HH:MM
> 项目目录：<当前工作目录>

## 背景

...

## 已完成的工作

- ...

## 当前状态

...

## 遗留问题 / 待办

- ...

## 关键决策与原因

- ...

## 注意事项

- ...
```

如果是追加更新，在文件末尾添加分隔：

```markdown
---

# 更新记录

> 更新时间：YYYY-MM-DD HH:MM
> 项目目录：<当前工作目录>

## 本次更新内容

...
```

### `/handoff list`

列出 `~/.claude/handoffs/` 目录下的所有交接文件。

#### 处理流程

1. 扫描 `~/.claude/handoffs/` 目录
2. 对每个 `.md` 文件，读取前几行提取标题和创建时间
3. 按修改时间倒序排列
4. 输出列表

#### 输出格式

```
## 交接文件列表

| # | 文件名 | 标题 | 创建时间 | 项目目录 |
|---|--------|------|----------|----------|
| 1 | foo.md | ... | 2026-04-11 | /path/to/project |
| 2 | bar.md | ... | 2026-04-10 | /path/to/project |

共 N 个文件，存储路径：~/.claude/handoffs/
```

如果目录为空或不存在：

```
## 交接文件列表

暂无交接文件。

存储路径：~/.claude/handoffs/
```

### `/handoff read <filename>`

读取指定交接文件并直接输出其内容。

#### 处理流程

1. **解析 filename**：取用户提供的 filename，若不以 `.md` 结尾则自动补上 `.md`
2. **读取文件**：读取 `~/.claude/handoffs/<filename>.md`
3. **直接输出**：将文件原始内容原封不动地输出，不做任何摘要或改写

如果文件不存在，输出：

```
文件不存在：~/.claude/handoffs/<filename>.md
```

### `/handoff del <filename>`

删除指定交接文件。此操作不可逆，**必须经过用户明确确认**才能执行。

#### 处理流程

1. **解析 filename**：取用户提供的 filename，若不以 `.md` 结尾则自动补上 `.md`
2. **检查文件是否存在**：
   - 文件不存在：直接输出提示，终止流程
     ```
     文件不存在：~/.claude/handoffs/<filename>.md
     ```
3. **向用户确认**：展示文件路径，明确告知将永久删除，要求输入确认
   ```
   即将删除文件：~/.claude/handoffs/<filename>.md
   此操作不可逆，文件内容将永久丢失。

   请输入 "确认删除" 以继续，或任意其他内容取消操作。
   ```
4. **检查确认内容**：
   - 用户未输入"确认删除"：输出"操作已取消。"，终止流程
   - 用户输入"确认删除"：执行删除
5. **执行删除**：使用 Bash 工具删除文件
6. **输出结果**：
   ```
   已删除：~/.claude/handoffs/<filename>.md
   ```
