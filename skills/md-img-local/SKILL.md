# SKILL: md-img-local（Markdown图片本地化）

## Summary

自动处理Markdown文档中的所有网络图片资源：
1. 识别所有HTTP/HTTPS网络图片链接
2. 批量下载到当前目录的`assets`文件夹
3. 自动添加基于原文档名的唯一前缀，彻底避免不同文档的图片重名覆盖
4. 自动替换原文件中的网络链接为本地相对路径
5. 保留图片原有格式和质量，无压缩或改动

## When to Use

在以下场景使用本 Skill：
1. 用户要求下载Markdown中的图片到本地
2. 用户需要将Markdown文档改为离线可访问版本
3. 用户担心网络图片链接失效，需要本地化备份
4. 多个Markdown文档需要统一管理图片资源，避免重名冲突
5. 用户明确提到"图片本地化"、"替换图片链接"、"下载图片"等关键词

## When Not to Use

以下场景不应使用本 Skill：
1. 用户要求保留原始网络图片链接，不需要本地化
2. Markdown文件中没有任何网络图片
3. 用户只需要提取图片链接，不需要下载和替换
4. 用户希望将图片上传到图床而不是保存到本地

## Success Criteria

只有当以下条件同时满足时，任务才算完成：
1. 所有网络图片都已成功下载到`assets`目录
2. 图片文件名都添加了对应文档的唯一前缀，无重名风险
3. 原Markdown文件中的所有网络图片链接已替换为本地相对路径
4. 替换后的链接可以正常访问，图片显示正常
5. 没有遗漏任何网络图片，也没有错误替换非图片链接
6. 原Markdown文档的其他内容没有被改动

## Inputs

输入通常包括：
- 要处理的Markdown文件路径
- 可选的参数：
  - 指定图片保存目录（默认：assets）
  - 指定文件名前缀（默认：使用原Markdown文件名）

## Outputs

标准输出：
1. 更新后的Markdown文件（原文件直接修改）
2. `assets/`目录下的所有本地化图片，命名格式：`[文档前缀]_[序号]_[原文件名]`

## Constraints

执行时必须遵守以下约束：
1. **不得修改Markdown文件的其他内容**，只能替换图片链接
2. **不得改动图片本身内容**，下载的图片必须和原网络图片完全一致
3. **必须保证文件名唯一性**，不同文档的图片不能重名
4. **不得遗漏任何网络图片**，必须全部处理
5. **不得覆盖本地已有的同名文件**，如果发现冲突要提示用户
6. 下载失败的图片要保留原链接，并提示用户下载失败

## High-Level Strategy

1. 先扫描Markdown文件，提取所有唯一的网络图片链接
2. 检查assets目录是否存在，不存在则自动创建
3. 对每个图片链接，生成唯一的本地文件名
4. 批量下载图片到本地
5. 逐行替换Markdown中的图片链接
6. 验证所有链接替换正确，图片可访问

## Procedure

### Step 1: 扫描Markdown文件
1. 读取目标Markdown文件内容
2. 使用正则提取所有`![alt](url)`格式的图片链接
3. 过滤出所有HTTP/HTTPS开头的网络链接
4. 去重相同URL的图片，避免重复下载

### Step 2: 准备本地存储
1. 检查当前目录下是否存在`assets`目录
2. 不存在则自动创建该目录
3. 提取原Markdown文件名作为图片前缀（去除特殊字符）

### Step 3: 下载图片
1. 遍历所有唯一的图片URL
2. 为每个图片生成唯一文件名：`[前缀]_[序号]_[原文件名]`
3. 下载图片到`assets/`目录，失败重试最多2次
4. 记录成功和失败的下载结果

### Step 4: 替换链接
1. 遍历原Markdown文件内容
2. 将每个网络图片URL替换为对应的本地相对路径
3. 只替换图片链接，不修改其他内容

### Step 5: 验证结果
1. 检查所有下载的图片是否存在，大小正常
2. 检查Markdown文件中的链接是否都已正确替换
3. 统计处理结果：总图片数、成功数、失败数

### Step 6: 反馈结果
向用户报告处理结果：
- 成功下载的图片数量
- 失败的图片链接（如有）
- 图片保存位置
- 提示用户可以离线访问文档了

## File Naming Conventions

### 图片文件名
```text
[文档前缀]_[序号]_[原文件名]
```
- 文档前缀：原Markdown文件名（去除空格、特殊字符）
- 序号：按图片出现顺序编号
- 原文件名：保留原URL中的文件名

### 示例
对于文档`某百万DAU游戏的服务端优化工作.md`中的第一张图片：
```text
百万DAU服务端优化_1_architecture.jpg
```

## Failure Modes

### 1. 下载失败
表现：部分图片URL无法访问或下载超时
处理：保留原链接，向用户报告失败的URL列表，建议手动处理

### 2. 文件名冲突
表现：本地已存在相同文件名的图片
处理：自动添加序号后缀，不覆盖已有文件，或提示用户确认

### 3. 错误替换非图片链接
表现：把普通HTTP链接当成图片链接替换了
处理：严格匹配`![alt](url)`格式，只替换图片链接

## Example Invocation

用户输入：
```text
处理一下这个markdown文件的图片，下载到本地
```
或
```text
/md-img-local 某百万DAU游戏的服务端优化工作.md
```

执行策略：
1. 扫描该Markdown文件，提取所有网络图片
2. 创建assets目录
3. 下载所有图片，添加唯一前缀
4. 替换原文件中的链接
5. 向用户报告处理结果

## Minimal Checklist

执行前后快速检查：
- [ ] 是否正确提取了所有网络图片链接
- [ ] 是否创建了assets目录
- [ ] 图片文件名是否添加了唯一前缀
- [ ] 所有图片是否下载成功
- [ ] 原Markdown文件中的链接是否都已替换
- [ ] 其他内容是否没有被改动
